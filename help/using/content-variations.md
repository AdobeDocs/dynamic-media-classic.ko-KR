# 콘텐츠 변형 관리{#managing-content-variations}

자산 변형이 게시되는 방법을 관리하려면 템플릿 집합을 사용합니다.

템플릿의 변형을 관리하려면 템플릿 집합을 만듭니다. 사이트에서 코드를 변경하지 않고 어떤 변형을 사용할지 제어할 수 있습니다. 이 방법을 사용하면 콘텐츠 관리자가 기술 지원을 통해 웹 코드의 URL을 변경하지 않고도 콘텐츠를 업데이트할 수 있습니다.

범용 URL은 설정된 순서에 따라 페이지의 템플릿 변형을 표시합니다. 템플릿 집합 목록의 상단에 있는 템플릿은 항상 게시됩니다.

목록에서 이미지 사전 설정 URL을 사용할 수 있습니다. 이미지 사전 설정 URL은 범용 URL과 유사하게 작동합니다. 이미지 사전 설정 URL이 두 개 이상 있을 수 있습니다.

**콘텐츠 변형을 관리하려면:**

1. **[!UICONTROL 빌드]** > **[!UICONTROL 템플릿 집합]**(으)로 이동합니다.
1. 빌더에서 템플릿을 선택한 다음 **[!UICONTROL 추가/미리 보기]**&#x200B;를 선택합니다.
1. 템플릿 속성을 수정하고 **[!UICONTROL 다른 이름으로 저장]**&#x200B;을 선택하여 다른 버전을 만듭니다.
1. 이름을 입력한 다음 **[!UICONTROL 저장]**&#x200B;을 선택합니다.

   에셋과 템플릿을 모두 게시해야 합니다.

1. [세부 사항] 페이지로 이동하여 URL 섹션에서 URL을 복사합니다.

템플릿을 새 위치로 드래그하여 재배열할 수 있습니다(예: 목록의 첫 번째 위치로). 새 순서를 전송하려면 다시 게시합니다.

>[!NOTE]
>
>필요한 경우 캐시를 지워 변경 사항을 확인합니다. 변경 사항은 캐시 주기를 통해 변경 사항이 처리된 후에만 웹 사이트에 표시됩니다.
